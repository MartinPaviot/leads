Flow complet du cold call — Elevay

De la constitution du TAM jusqu'à la mise à jour automatique des deals. Chaque étape indique le provider réel et le fichier qui le porte. Orchestration par Inngest, stockage Postgres / Drizzle.

Capture des comptes (TAM)lib/voice/source-prospects.ts · lib/discovery/*

L'ICP du tenant alimente une recherche d'entreprises. Le TAM "vivant" se remplit par deux voies, dédupliquées par domaine.

APOLLO Recherche d'organisations

searchOrganizations() depuis les filtres ICP (secteur, taille, géo, revenu, tech). Insère les nouvelles companies.

SIRENEPAPPERSZEFIX Discovery multi-source

Sources nationales keyless : SIRENE/Pappers (FR), Zefix (CH). Provenance + bridge de résolution de domaine, passées par la queue d'approbation.

Apollo masque les firmographics au search (industry/size/revenue NULL) → enrichOrganization(domain) derrière pour remplir la fiche compte.

Capture des contactslib/voice/source-prospects.ts

Pour chaque compte, on cherche les personnes qui matchent la persona (séniorité + intitulés de rôle), dédupliquées par email.

APOLLO Recherche de personnes

searchPeople({ domain, seniorities, titles }) → insère les contacts (nom, titre, LinkedIn, email/phone bruts).

EVENT Déclenche l'enrichment

Chaque contact inséré émet contact/created (Inngest) pour résoudre téléphone + email vérifié — la liste d'appel a besoin d'un numéro.

Enrichment téléphone & email — waterfall geo-routéelib/providers/contact-enrichment/waterfall.ts

Les providers tournent par priorité boostée selon le pays du prospect. On s'arrête dès qu'on a un mobile joignable + un email utilisable.

APOLLO prio 10

Reveal par id : last_name, LinkedIn, email vérifié, numéros. Sert aussi de LinkedIn-source pour Kaspr.

KASPR prio 20 · FR

Leader mobile FR. Boosté devant Apollo pour les +33.

LUSHA prio 30 · CH/EU

Seule source mobile suisse qu'on a. Cap 100/jour, enrichi par vagues.

Routage : FR → Kaspr → Lusha → Apollo  ·  CH → Lusha → Apollo → Kaspr (Kaspr est FR-only)  ·  US/autre → Apollo → Kaspr → Lusha.  ZELIQ en complément asynchrone (webhook). Chaque provider est gaté par sa clé : sans clé, il est sauté.

Call Mode — déclenchement de l'appelapi/calls/start · lib/voice/twilio.ts

Le contact (avec téléphone) entre dans la liste du jour. À l'appel : réservation d'une ligne calls, garde-fous, puis ouverture de la jambe sortante.

GARDE-FOUS avant numérotation

Sélection du numéro émetteur (pool tenant), check DNC, quiet hours (fuseau prospect), usage cap (4000 min/siège).

TWILIO Outbound + WebRTC

createCall() place la jambe ; un JWT capability attache le micro du navigateur via le Voice SDK. TwiML connecte et ouvre un Media Stream.

Transcript live + coachinglib/voice/deepgram-bridge.ts · coaching-tap.ts

L'audio de l'appel est transcrit en direct ; chaque réplique du prospect peut déclencher une carte de coaching.

TWILIO Media Stream

Frames μ-law 8 kHz mono (base64) poussées sur le WebSocket — aucun resampling.

DEEPGRAM Nova-3 live

encoding=mulaw, diarisation, FR. Les chunks finals sont ajoutés à calls.transcript (jsonb), poussés à l'UI via SSE.

SONNET 4.6 Coaching tap

Chunk prospect → classifieur d'objection → carte de réponse suggérée (calls.coachingCards), en direct.

Analyse du transcript (post-appel)inngest/calls-post-process.ts

Le webhook recording-status de Twilio émet calls/post-process. Le worker Inngest transforme la conversation en données structurées.

DEEPGRAM Fallback batch

Si le stream live a lâché (transcript vide), transcription batch sur l'enregistrement Twilio.

SONNET 4.6 → fallback gpt-4o-mini · Extraction structurée

generateObject(callNotesSchema) : outcome, summary, sentiment, buyingSignals (budget/timeline/stack/objections), action items, callback. Persisté sur calls.

En parallèle : activité call_completed (via seam d'approbation auto/review), indexation RAG du transcript pour le coaching, détection DNC sur mots-clés.

Mise à jour des deals — CRM auto-looplib/voice/post-call-crm.ts

L'outcome route l'état du CRM que le rep aurait saisi à la main. Idempotent : un seul deal ouvert par compte, tâches taguées [call:<id>].

meeting_booked / connected+positif

Crée ou fait avancer un deal (stage demo/qualification). Valeur ← budget, close date ← timeline. Patch des signaux d'achat.

callback / action items

Crée des tasks : rappel daté à la demande, items d'action (owner + échéance), priorité high/medium.

not_interested

Ferme le deal ouvert en lost avec lostReason + lostCallId.

CONTEXT GRAPH Mémoire client

ingestEpisode() : épisode bi-temporel (qui, quand, signaux) interrogeable par le chat + l'intelligence.

CADENCE Campagne

recordCallOutcomeForCampaigns() : no-answer/busy → re-queue (8x/15j) ; connecté/meeting → fin de cadence.

CONTACT Disposition

Stamp lastCall (outcome, sentiment, date, callId) sur le contact.
