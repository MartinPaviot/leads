# Archived migrations (pre baseline-squash)

These 81 `.sql` files were the migration history up to `origin/main` @ `7f02b154`.
They could **not** replay on a fresh database — confirmed by iterative testing:
ordering bugs, a `drizzle-kit generate` snapshot (`0012_wandering_hemingway`)
duplicating granular migrations (CREATE + FK + index), runtime-only tables
(`embeddings`, `custom_records`) ALTERed before creation, a PL/pgSQL ambiguity in
`0038`, and an RLS fidelity gap (policies on tables the replay left without
`tenant_id`). See `_research/migration-baseline-squash-plan.md`.

They were squashed into `../0000_baseline.sql`, generated by `pg_dump --schema-only
--schema=public` from the live `leadsens-dev` (prod) database — the canonical schema
(121 tables, 84 RLS policies, 434 indexes, 19 enums, `vector` extension).

`scripts/apply-migrations.ts` reads only `drizzle/*.sql` (not this `_archive/`), so
these no longer run. Kept for historical reference / git blame.

Pre-existing databases were reconciled with `scripts/baseline-reconcile.ts`
(records the baseline as already-applied without re-running it).
