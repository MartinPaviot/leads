## Dependency / ordering notes
- **Tasks 1–3 (extractions + helpers + refs) before Task 4** — the actions' `run`s call the extractions and read the refs; doing them first means Task 4 only *declares*, never *duplicates*.
- **Task 4 (list) and Task 5 (detail) are largely independent** but Task 5 also does the detail extractions (`startCallResult`/`reviewCallIntel`); both depend on the refs (Task 3). They can be parallelized after Task 3.
- **Task 7 depends on 4 + 5** (it tests both paths). **Task 8 depends on 4 + 5** being green. **Task 9 is the final gate.**
- **CLE-05 interaction:** `bulkEnrich`/`bulkFindMobile`/`bulkMerge`/`bulkRestore`/`scoreAll`/`createContact`/`updateField`/`reassignOwner`/`approveCallIntel`/`dismissCallIntel` are `confirm:"risky"`, `bulkDelete`/`call` are `confirm:"always"` → they reach CLE-05's confirm card before `run` (`call` shows a "Sends externally" badge; the two credit ones show "Uses credits"). `applyFilter`/`smartSearch`/`selectAll`/`openImport`/`openSmartImport`/`sendEmail`/`suggestReply` are `confirm:"never"` → CLE-03 runs them immediately. CLE-08 does not render any card itself; it only declares the metadata that drives the gate.
- **THE FILE-PICKER HUMAN-BOUND BOUNDARY (design §2) is the headline novelty.** `openImport`/`openSmartImport` are non-mutating "navigate/open" actions: the agent opens the picker/modal, the **human chooses the file**. There is no file param and no reachable upload route from the registry — the boundary is *absence of mechanism*, not a check. This is the README §2 exclusion realized for the first time; the prompt/UX wording ("I can't pick the file for you") is set here and reused by CLE-14 (meetings transcript / proposals template upload). It changes no CLE contract.
- **CLE-11 (audit/undo):** these actions declare `reversible` honestly (soft-delete→restore, re-score, field revert) — **except `call`, which is `reversible:false`** (a placed call cannot be unplaced). The undo *mechanism* + `tool_call_events` logging (and any "undo window" for the outbound call / credit spend) is CLE-11 — out of scope here.
- **Bulk/cross-page ops stay headless** (README §3.6): "enrich every contact at French accounts", "delete all contacts with no email across the base" act beyond the current selection/loaded list → route to headless contact tools, not these page actions. The system-prompt heuristic (CLE-04 §2.9) already teaches this; CLE-08 adds nothing for it.
- **Deferred (CLE-14 if wanted):** per-row inline `enrichSingle` (`:303-311`) and single-row delete (`:1097`) — the agent's path is select + bulk, or headless; the merge page's own `mergeGroup` commit (`merge/page.tsx:131-171`).
